## Project Overview

PHP 8.x web application for managing swimming competition start lists and live results for Olimpijczyk Proszówki. No framework, no database — all data is stored as JSON files.

## Running and Testing

```bash
# Run a PHP file locally (CLI)
php includes/functions.php

# Run the cron result-fetcher manually
php cron/check_results.php

# Generate a bcrypt password hash for admin login
php -r "echo password_hash('password', PASSWORD_BCRYPT);"
```

No automated tests are configured. Verify changes in a browser or via CLI.

## Architecture

### Data Storage (JSON files, no database)

- `zawody/*.json` — one file per competition; required field: `bloki` (array of race blocks)
- `zawodnicy/*.json` — per-athlete race history, auto-generated from result fetching
- `zapowiedzi.json` — competitions without a start list ("coming soon")
- `live_config.json` — currently active live competition config

### Key includes

| File | Role |
|------|------|
| `includes/config.php` | All constants: `BASE_URL`, `ZAWODY_DIR`, `ZAWODNICY_DIR`, `ADMIN_PASSWORD_HASH`, `CSS_VERSION`, `RESULT_DELAY_SECONDS` |
| `includes/functions.php` | Competition/announcement CRUD, `h()` (HTML escape), `slugify()`, CSRF helpers, `format_konkurencja()` |
| `includes/athlete.php` | Athlete profile load/save/dedup — `save_athlete_result()` deduplicates by competition+date+event_nr |
| `includes/result_fetch.php` | Fetches `ResultList_{nr}.pdf` from livetiming.pl, updates competition JSON and athlete profiles |
| `includes/pdf_extract.php` | PDF text extraction: tries `pdftotext` (poppler-utils) first, falls back to pure PHP FlateDecode/BT-ET parser |
| `includes/auth.php` | Admin session guard — include at the top of every protected page |

### Result fetching pipeline

1. Admin sets a livetiming.pl `index.html` URL + competition JSON file via `admin/live.php` → saved to `live_config.json`
2. `api/fetch_result.php` (AJAX POST, same-origin only) or `cron/check_results.php` calls `process_pending_results()`
3. For each entry older than `RESULT_DELAY_SECONDS` with no result: downloads `ResultList_{event_nr}.pdf`, extracts athlete name + time + points
4. Writes `czas_result`, `punkty`, `result_fetched: true`, `result_fetched_at` back into the competition JSON
5. Updates/creates the athlete profile in `zawodnicy/`

### Public pages

- `index.php` — competition grid (JSON files + announcements, sorted by mtime)
- `lista_startowa.php?f=filename` — start list grouped by blocks
- `wyniki.php?zawody=filename` — live results with JS polling (60 s) and auto-refresh meta (120 s)

### Admin panel (`/admin/`, session auth)

All pages require `includes/auth.php`. All POST forms include a CSRF token generated by `csrf_token()` and verified by `csrf_verify()`.

## Configuration

Edit `includes/config.php` for:
- `ADMIN_PASSWORD_HASH` — bcrypt hash of admin password
- `BASE_URL` — set to e.g. `'/swim'` if not deployed at web root (default: `''`)
- `CSS_VERSION` — bump on CSS changes to bust browser cache
- `RESULT_DELAY_SECONDS` — wait before fetching results after a race start (default: 300)

## Security Patterns

- All output through `h()` (`htmlspecialchars` with ENT_QUOTES/UTF-8)
- File access restricted to `zawody/` via `safe_json_path()` (basename + alphanumeric/dash/underscore whitelist)
- `/includes/` and `/cron/` are blocked by `.htaccess` from direct HTTP access
- API endpoint validates same-origin via `HTTP_ORIGIN` vs `HTTP_HOST`

## CSS Cache-Busting

After modifying any CSS file, increment `CSS_VERSION` in `includes/config.php`. The version is appended as `?v=N` to stylesheet `<link>` tags.
